Micrograd 是什么

Micrograd 是一个标量值 autograd 引擎。Autograd（automatic gradient，自动梯度）本质上就是实现了反向传播的系统；反向传播是一种高效算法，用于计算损失函数对网络中每一个权重的梯度。一旦获得这些梯度，就可以迭代地调整权重以最小化损失并提升准确率。这一过程位于每一个现代深度学习库（如 PyTorch 或 JAX）的数学核心。 [00:00:32]

Micrograd 的 API 最好通过示例说明。两个标量输入 a = -4 和 b = 2 被包装成 Value 对象，并组合成更大的表达式：

a = Value(-4.0)
b = Value(2.0)
c = a + b
d = a * b + b**3
c += c + 1
c += 1 + c + (-a)
d += d * 2 + (b + a).relu()
d += 3 * d + (b - a).relu()
e = c - d
f = e**2
g = f / 2.0
g += 10.0 / f

支持的操作包括加法、乘法、常数幂运算、取负、ReLU、除以常数等。在构建表达式的过程中，Micrograd 会在 Value 节点之间维护指针，从而隐式地记录完整的计算图(computational graph)。 [00:01:34]

神经网络只是数学表达式

上面的表达式是任意的，其目的仅在于展示 Micrograd 支持的操作。真正重要的是，神经网络本身也是同类的数学表达式。网络以数据和权重为输入，执行一系列加法、乘法与非线性变换，最终输出预测值或损失值。反向传播并非神经网络所特有——它对任何可微的数学表达式都适用。神经网络只是恰好让这套机制发挥作用的其中一类表达式。 [00:04:42]

为什么是标量？

Micrograd 在单个标量上运算，将网络拆解到单个数字和原始操作的原子级别。这并不是生产系统的做法；真正的框架将标量打包成 n 维张量(tensor)，以便操作能够利用并行计算。然而，数学原理并未改变。使用标量消除了张量(tensor) bookkeeping 的干扰，使底层的反向传播和链式法则变得清晰透明。 [00:05:45]

讲师定义了一个简单的二次函数作为贯穿全节的示例：

import numpy as np
import matplotlib.pyplot as plt

def f(x):
    return 3*x**2 - 4*x + 5

f(3.0)  # returns 20.0

为了观察其形状，他在区间 -5 到 5 上以 0.25 为步长采样并绘制结果，确认了预期的抛物线。点 (3.0, 20.0) 位于曲线右侧的上升段 [00:08:18]。

他没有进行符号微分，而是转向导数的极限定义：

$$ \frac{df}{dx} = \lim_{h \to 0} \frac{f(x + h) - f(x)}{h} $$

其含义很直接：将输入微小地扰动一个 h，观察输出变化了多少，再用扰动的大小进行归一化。结果就是该点处函数的斜率，即敏感度 [00:10:22]。

选取一个很小但有限的 h = 0.001，在 x = 3.0 处的数值近似为：

h = 0.001
x = 3.0
slope = (f(x + h) - f(x)) / h  # approximately 14.0

由于该点位于抛物线的上升部分，将 x 向上微增会提高 f(x)。斜率为正，其大小告诉我们增长的程度 [00:11:26]。一个快速的符号验证确认了结果：3x^2 - 4x + 5 的导数是 6x - 4，在 x = 3 处取值为 14。

同样的步骤在 x = -3.0 处得到一个负斜率，约为 -22。在这里，对输入施加一个小的正向扰动会使函数值下降，因此敏感度为负 [00:13:00]。

最后，讲师指出每条抛物线都有一个最小值点，在该点处斜率恰好平坦为零。对于这个函数，最小值出现在 x = 2/3。在那个精确点上，输入的无穷小变化不会产生一阶输出变化，这是驻点(stationary point)的标志 [00:13:30]。

将导数直觉扩展到多输入函数

单输入函数是一个有用的起点，但真正的神经网络同时依赖许多参数。在构建处理那些庞大表达式的机制之前，我们需要看看导数的概念如何从一个输入扩展到多个输入。关键洞见在于，我们仍然可以问："如果我只微调一个输入，输出会发生什么变化？"——前提是我们保持所有其他输入不变。

为了探索这一点，考虑一个简单的函数，它有三个标量输入和一个输出：

a = 2.0
b = -3.0
c = 10.0
d = a * b + c

在这些值下计算 d 得到 4.0。问题是：d 对 a、b 和 c 各自的敏感度如何？[00:14:03]

我们可以通过引入一个微小的增量 h，重新计算只改变一个输入后的 d，并测量变化来数值地回答这个问题。例如，为了求关于 a 的导数：

h = 0.0001

# baseline
d1 = a * b + c

# bump a by h
a += h
d2 = a * b + c

slope = (d2 - d1) / h

在运行代码之前，值得先推理一下符号。因为 b 是负数（-3.0），稍微增加 a 意味着用一个稍微大一点的正数乘以 -3.0，这会从总数中减去更多。因此 d 应该减小，斜率应该是负的。[00:16:10] 运行计算证实了这一点：d 从 4.0 下降到大约 3.9996，得到的斜率为 -3.0。从解析上看，这是合理的，因为对 a * b + c 关于 a 求导后剩下 b，而 b 是 -3.0。[00:17:16]

对 b 重复实验则讲述了一个不同的故事。由于 a 是正数（2.0），将 b 向上微调会增加乘积 a * b，因此 d 增加。数值斜率结果为 2.0，与解析导数 ∂d/∂b = a = 2.0 相符。[00:17:48]

最后，将 c 增加 h 会使 d 改变完全相同的量，因为 c 仅以加法项的形式出现。斜率为 1.0，即解析导数 ∂d/∂c = 1。[00:18:18]

这三个值——-3、2 和 1——分别是 d 关于 a、b 和 c 的偏导数(partial derivatives)。每一个都隔离了单个输入的影响，同时其余输入保持不变。这一原理，当扩展到数千或数百万个输入时，正是基于梯度的学习成为可能的原因。下一步是构建能够表示和区分神经网络中庞大表达式的数据结构。[00:18:48]

教会 Python 使用 + 和 *

默认情况下，Python 不知道如何对自定义对象进行加法或乘法运算。我们定义 __add__ 和 __mul__ 这两个 dunder 方法，让表达式如 a + b 和 a * b 能够自然工作。在内部，a + b 被翻译成 a.__add__(b)，a * b 被翻译成 a.__mul__(b)。每个运算符返回一个全新的 Value，包装底层浮点运算的结果。

class Value:
    def __init__(self, data):
        self.data = data

    def __repr__(self):
        return f"Value(data={self.data})"

    def __add__(self, other):
        out = Value(self.data + other.data)
        return out

    def __mul__(self, other):
        out = Value(self.data * other.data)
        return out

现在我们可以编写看起来像普通数学的表达性代码：

a = Value(2.0)
b = Value(-3.0)
c = Value(10.0)
d = a * b + c   # (2.0 * -3.0) + 10.0 == 4.0

表达式 a * b + c 从左到右求值：首先 a.__mul__(b) 产生 Value(data=-6.0)，然后该结果的 __add__ 被调用并传入 c，得到 Value(data=4.0)。 [00:20:21] [00:22:26]

追踪来源：子节点与操作

仅有算术运算不足以实现反向传播。我们需要重建整个表达式图，这意味着每个 Value 都必须记住是哪些数值创建了它，以及通过哪个操作。我们在构造函数中引入两个新字段：_prev，一个子节点集合；_op，一个记录操作名称的字符串。

class Value:
    def __init__(self, data, _children=(), _op=''):
        self.data = data
        self._prev = set(_children)
        self._op = _op

    def __repr__(self):
        return f"Value(data={self.data})"

    def __add__(self, other):
        out = Value(self.data + other.data, (self, other), '+')
        return out

    def __mul__(self, other):
        out = Value(self.data * other.data, (self, other), '*')
        return out

叶节点——直接从标量创建的数值——具有空的 _prev 和空的 _op 字符串。内部节点携带完整的来源信息。对于表达式 d = a * b + c，检查 d._prev 可以揭示两个中间值（a*b 和 c），而 d._op 告诉我们最后一步是加法。 [00:22:56] [00:24:29]

有了这个连接组织，Value 对象不再仅仅是一个数字；它是一个活跃表达式图中的节点。下一步是在图变大时将其可视化。

手动反向传播：一个简单表达式

反向传播(backpropagation)是计算计算图(computational graph)中每个中间值如何影响最终输出的过程。在神经网络中，输出通常是损失(loss)函数，而叶节点是网络的权重(weight)和输入数据。我们通常不需要关于数据本身的梯度(gradient)，因为数据是固定的；我们需要的是关于权重的梯度，之后我们会用它们通过梯度下降(gradient descent)来更新权重。 [00:29:39]

为了跟踪这些导数，我们在 Value 类中添加一个 gradient 字段。它存储最终输出（这里称为 l）关于该 Value 的导数。默认情况下，gradient 初始化为 0.0，表示最初假设每个值对输出都没有影响。 [00:30:41]

反向传播从图的末端开始。l 关于自身的导数是 1.0，因为 l 的一个微小变化 h 会恰好使 l 改变 h。我们可以通过将一个叶节点增加一个小的 h，测量 l 的变化，并除以 h 来数值验证这一点。为了避免污染全局命名空间，我们将这个检查包装在一个局部函数中：

def lol():
    h = 0.001
    a = Value(2.0, label='a')
    b = Value(-3.0, label='b')
    c = Value(10.0, label='c')
    f = Value(-2.0, label='f')
    e = a * b; e.label = 'e'
    d = e + c; d.label = 'd'
    l = d * f; l.label = 'l'

    l1 = l.data
    # 将 a 增加 h
    a.data += h
    l2 = l.data
    print((l2 - l1) / h)

运行这段代码得到 l 关于 a 的导数。对于基础情况，我们手动设置 l.gradient = 1.0。 [00:32:12]

接下来，考虑节点 l = d * f。局部导数很简单：如果 l = d * f，那么 dl/dd = f 且 dl/df = d。这可以从导数的极限定义推导出来，其中 d*f 项相消后留下 f（或 d）。因此，d.gradient = f.data = -2.0 且 f.gradient = d.data = 4.0。一个快速的数值检查确认了这些值。 [00:34:51]

反向传播的精髓在于通过中间节点应用链式法则(chain rule)。以加法节点 d = c + e 为例。我们已经知道 dl/dd = -2.0。局部地，一个和关于其任一输入的导数是 1.0（同样可以通过极限定义验证）。链式法则表明：

$$ \frac{\partial l}{\partial c} = \frac{\partial l}{\partial d} \cdot \frac{\partial d}{\partial c} $$

由于 dd/dc = 1.0，梯度直接路由通过：c.gradient = dl/dd * 1.0 = -2.0。由对称性，e.gradient 也是 -2.0。直观上，一个加号节点将传入的梯度平均分配给其子节点，因为其局部导数都是 1。 [00:38:27]

最后，我们递归到乘法节点 e = a * b。我们知道 dl/de = -2.0。局部梯度是 de/da = b = -3.0 和 de/db = a = 2.0。再次应用链式法则：

a.grad = dl_de * de_da = -2.0 * -3.0 = 6.0
b.grad = dl_de * de_db = -2.0 *  2.0 = -4.0

数值验证确认 a.gradient = 6.0 和 b.gradient = -4.0。 [00:47:20]

本质上，手动反向传播从输出到叶节点迭代遍历整个图。在每个节点，我们取损失关于该节点输出的导数（我们已经知道）并将其乘以该节点操作关于每个输入的局部导数。这个乘积随后被赋值为该输入的梯度，允许信号递归地向后流过整个图。 [00:50:30]

使用梯度进行单步优化

既然已经计算出了图中每个节点的梯度，我们就可以利用它们来影响最终输出。原理很简单：每个叶节点的梯度精确地告诉我们，对该节点进行微小改动会如何影响最终的损失 L。通过让每个叶节点沿着其梯度的方向移动，我们就能增大最终的值 L。

这是基于梯度的优化的核心思想。这里演示的过程是单步操作，之后它会扩展为迭代训练循环。同时微调所有输入，充分利用了反向传播获得的全部信息。

单步梯度上升

回顾叶节点 a、b、c 和 f。它们相对于 L 的梯度已经计算完毕。为了让 L 增大（变得不那么负），用每个叶节点的梯度乘以一个很小的比例后加到自身：

# Step size (learning rate)
step = 0.01

# Nudge leaf nodes in the direction of their gradients
a.data += step * a.grad
b.data += step * b.grad

自动反向传播：backward() 与拓扑排序

存储局部梯度：为每个操作添加 backward 函数

每个 Value 节点都需要知道如何将梯度反向路由经过自身。我们在每个节点上引入一个 _backward 闭包，它捕获了创建该节点的操作的局部链式法则逻辑。默认情况下，这个闭包什么都不做，这对于没有父节点需要更新的叶节点来说是正确的行为。对于加法、乘法和 tanh 等操作，我们定义一个自定义闭包，它接收 out.gradient 并使用相应的局部导数将其传播到 self.gradient 和 other.gradient。

这种设计是实现自动微分模块化的关键一步：每个节点负责自身反向传播中的一个小部分，因此我们不再需要一个大型的梯度函数来理解图中的每一个操作。

默认的 _backward 闭包

在 Value 的构造函数中，我们将 _backward 初始化为一个空操作函数：

class Value:
    def __init__(self, data, _children=(), _op=''):
        self.data = data
        self.grad = 0.0
        self._backward = lambda: None   # 默认为空操作
        self._prev = set(_children)
        self._op = _op

加法的 backward

当我们创建 out = self + other 时，和相对于每个输入的局部导数是 1.0。根据链式法则，self.gradient 和 other.gradient 各自接收 out.gradient 乘以该局部导数：

def __add__(self, other):
    other = other if isinstance(other, Value) else Value(other)
    out = Value(self.data + other.data, (self, other), '+')

    def _backward():
        self.grad += 1.0 * out.grad
        other.grad += 1.0 * out.grad
    out._backward = _backward

    return out

由于局部导数是 1.0，out.gradient 只是被复制到两个父节点的梯度中。这与我们之前手动追踪梯度时观察到的行为一致。[01:10:35]

乘法的 backward

对于 out = self * other，相对于 self 的局部导数是 other.data，相对于 other 的局部导数是 self.data。backward 闭包变为：

def __mul__(self, other):
    other = other if isinstance(other, Value) else Value(other)
    out = Value(self.data * other.data, (self, other), '*')

        self.grad += other.data * out.grad
        other.grad += self.data * out.grad
    out._backward = _backward

这里链式法则表现为：在将 out.gradient 累加到每个父节点之前，先将其按兄弟操作数的数据值进行缩放。[01:12:11]

Tanh 的 backward

对于 out = tanh(self)，回顾 $\tanh(x)$ 的导数是 $1 - \tanh^2(x)$。由于 out.data 已经保存了 $\tanh(x)$，我们可以将局部导数简洁地写为 $1 - \text{out.data}^2$：

def tanh(self):

    def _backward():
        self.grad += (1 - t**2) * out.grad
    out._backward = _backward

    return out

拓扑排序：自动化逆序遍历

到目前为止，我们一直在每个中间节点上手动调用 _backward()。对于微小表达式这还能工作，但它很脆弱且无法扩展。我们真正需要的是一种遍历图的方式，使得每个节点在向自己的父节点传播梯度之前，先从所有依赖于它的节点接收梯度。这种顺序正是拓扑排序所提供的。

有向无环图（DAG, directed acyclic graph）的拓扑排序将节点排成一条线，使得每条边都从左指向右。在我们的计算图中，这意味着所有依赖项都出现在依赖于它们的节点之前。一旦有了这样的顺序，我们就可以逆序遍历它，并在每个节点上自动调用 _backward()。

        visited = set()

            if v not in visited:
                visited.add(v)
                for child in v._prev:
                    build_topo(child)
                topo.append(v)
        build_topo(self)

        for node in reversed(topo):
            node._backward()

Bug：赋值覆盖了之前的梯度

考虑一个最小示例：

a = Value(3.0)
b = a + a
b.backward()

这里 a 参与了两次加法，因此计算图中包含两条从 a 到 b 的边。在反向传播期间，b 是一个加法节点，因此它的局部规则将 self.gradient 设为 1.0，other.gradient 设为 1.0。问题在于 self 和 other 指向的是同一个 Value 对象（a）。第一次赋值将 a.gradient 设为 1.0，第二次赋值又把它覆盖为 1.0。因为梯度是赋值而非累积，最终结果停留在 1.0，而 b 对 a 的真实导数应该是 2。

一个稍复杂的表达式能让这个模式更清晰：

a = Value(2.0)
b = Value(3.0)
d = a * b
e = a + b
f = e * d
f.backward()

在这个图中，a 和 b 各出现了两次。反向传播时，梯度从 f 正确地流向 e 和 d，但当 d.backward() 在 e.backward() 之后运行时，它会覆盖 e 已经写入 a 和 b 的梯度。每当一个变量被使用多次时，同样的覆盖就会发生。到目前为止测试的每个表达式恰好让每个变量只用了一次，这就是为什么这个 bug 一直没被发现。

修复：用 += 累积

多变量链式法则（multivariate chain rule）指出，当一个变量流入多个下游操作时，总梯度是沿每条路径的梯度之和。因此，梯度必须累积，而不是赋值。将 _backward lambda 中的每一次梯度更新从 = 改为 += 即可解决问题：

# 在 Value.__add__ 中
out._backward = lambda: (
    self.grad += out.grad,
    other.grad += out.grad,
)

# 在 Value.__mul__ 中
out._backward = lambda: (
    self.grad += other.data * out.grad,
    other.grad += self.data * out.grad,
)

# 在 Value.tanh 中
out._backward = lambda: (
    self.grad += (1 - out.data**2) * out.grad,
)

因为所有 gradient 字段都初始化为 0.0，所以每次反向传播的贡献都会叠加在之前的值之上。回到 a + a 的例子，a.gradient 现在从 0 开始，从第一条边接收 1.0，再从第二条边接收另一个 1.0，得到正确的值 2.0。同样，在 f = e * d 表达式中，e.backward() 写入 a 和 b 的梯度在 d.backward() 随后添加自己的贡献时仍然保留，从而得到正确的最终梯度。

下一个基本操作是自然指数。与 tanh 类似，exp 是一个一元函数：它取出原始标量，应用 math.exp，并将结果包装在一个新的 Value 中。

    def exp(self):
        x = self.data
        out = Value(math.exp(x), (self,), 'exp')
        def _backward():
            self.grad += out.data * out.grad
        out._backward = _backward
        return out

$e^x$ 的局部导数就是 $e^x$ 本身。因为 out.data 已经保存了 $e^x$，反向传播只需将 out.data 乘以传入的梯度 out.gradient 并累加到 self.gradient 中。这就是链式法则的实际应用。 [01:31:09]

幂运算：将 Value 提升到常数次幂

除法是更一般操作的特殊情况：将一个值提升到常数次幂。如果 $a / b$ 可以重写为 $a \cdot b^{-1}$，那么为常数 $k$ 实现 pow(k) 就能立即免费得到除法。

    def __pow__(self, other):
        assert isinstance(other, (int, float)), "only supporting int/float powers"
        out = Value(self.data**other, (self,), f'**{other}')
        def _backward():
            self.grad += (other * self.data**(other - 1)) * out.grad
        out._backward = _backward
        return out

微积分中的幂法则指出

$$\frac{d}{dx} x^n = n \cdot x^{n-1}.$$

这里 $n$ 是常数 other，$x$ 是 self.data。因此局部导数为 other * self.data**(other - 1)，通过上游梯度进行链式求导就得到了上面的完整表达式。 [01:34:13]

Neuron（神经元）

单个神经元接收固定数量的标量输入，计算加权和，加上偏置，然后将结果传入非线性激活函数。在构造函数中，神经元为每个输入存储一个权重，初始化为 -1 到 1 之间的均匀分布，以及一个控制其整体触发阈值的偏置。

class Neuron:
    def __init__(self, nin):
        self.w = [Value(random.uniform(-1, 1)) for _ in range(nin)]
        self.b = Value(random.uniform(-1, 1))

    def __call__(self, x):
        act = sum((wi * xi for wi, xi in zip(self.w, x)), self.b)
        out = act.tanh()
        return out

前向传播就是熟悉的 $w \cdot x + b$ 后接 $\tanh$。Python 的 sum 接受可选的 start 参数，因此将偏置作为初始累加值传入，避免了循环后再显式相加。调用神经元只需 n(x)，Python 会将其路由到 __call__。 [01:44:12]

Layer（层）

层不过是一组相互独立的神经元，每个神经元接收相同的输入向量，但产生各自的标量输出。由于神经元之间彼此独立，层可以将它们存储在列表中，并在前向传播时并行计算。

class Layer:
    def __init__(self, nin, nout):
        self.neurons = [Neuron(nin) for _ in range(nout)]

    def __call__(self, x):
        outs = [n(x) for n in self.neurons]
        return outs

创建一个具有两个输入和三个输出的层会得到一个包含三个 Value 对象的列表，每个神经元一个。同一层内部的神经元之间没有连接；全连接只发生在层与层之间。 [01:47:55]

MLP（多层感知机）

MLP 将层按顺序链式连接：一层的输出成为下一层的输入。构造函数接收输入维度，以及一个指定后续每层神经元数量的列表。然后遍历相邻的尺寸对来构建连接层。

class MLP:
    def __init__(self, nin, nouts):
        sz = [nin] + nouts
        self.layers = [Layer(sz[i], sz[i+1]) for i in range(len(nouts))]

    def __call__(self, x):
        for layer in self.layers:
            x = layer(x)
        return x[0] if len(x) == 1 else x

例如，一个具有三个输入、两个各含四个神经元的隐藏层、以及一个单输出的 MLP 可实例化为 MLP(3, [4, 4, 1])。__call__ 中最后的便捷处理会将单元素列表解包，使单输出网络直接返回标量 Value，而不是包含它的列表。 [01:49:28]

然而，表面上良好的结果隐藏了一个严重且极其常见的 bug：梯度在每一步之间从未被重置为零。每次调用 .backward() 都会对 p.gradient 使用 +=，因此梯度会无限累积。第二步把新的梯度加在旧的上面，第三步再加一次，依此类推。这人为地放大了有效步长，使收敛看起来比实际更快。 [02:10:23]

修复方法是在每次反向传播前显式地将梯度清零，正如 Value 构造器初始化 gradient = 0 一样：

for k in range(20):
    # zero gradients
    for p in n.parameters():
        p.grad = 0.0

    # forward, backward, update
    ypred = [n(x) for x in xs]
    loss = sum((yout - ygt)**2 for yout, ygt in zip(ypred, ys))
    loss.backward()
    for p in n.parameters():
        p.data -= 0.05 * p.grad

经过这一修正，下降过程更慢、更可控，但在足够的步数后仍然能达到低损失和准确的预测。之前带 bug 的运行之所以成功，只是因为数据集极小且问题过于简单；在更难的任务上，累积的梯度会使训练失稳。 [02:12:28]

engine.py 中的 Value 类包含与讲座中构建的相同要素：一个 data 标量、一个 gradient 累加器、一组 _prev 子节点，以及创建该节点的 _op。它实现了加法、乘法和标量幂运算，反向传播也完全一致。提供的非线性激活是 ReLU，而非 tanh：

def relu(self):

    def _backward():
        self.grad += (out.data > 0) * out.grad

在 micrograd 中，tanh 的反向传播实现如下：

# self.data holds tanh(x)
# out.grad holds upstream gradient
self.grad += (1 - self.data**2) * out.grad

这就是链式法则的实际应用：tanh 的局部导数是 $1 - \tanh^2(x)$，再乘以来自后续层的传入梯度。问题在于 PyTorch 是否编码了同样的公式。[02:20:53]

在 PyTorch 仓库中搜索 "tanh" 会在 400 多个文件中返回大约 2,800 条结果。代码库之所以庞大，是因为它必须支持多种设备、数据类型和分发机制。在缩小搜索范围后，相关的核函数出现在两个地方：一个用于 CPU，一个用于 CUDA。[02:21:23]

CPU 核函数位于一个处理许多逐元素操作的文件内部。在去掉复数和 bfloat16 类型的分支后，核心逻辑归结为：

a * (1 - b * b)

这里 b 是 tanh 的输出，a 是上游梯度。这正是 $1 - \tanh^2(x)$ 乘以 grad_output，与 micrograd 的代码逐行对应。CUDA 核函数更加紧凑，用一行设备代码表达了同样的计算。[02:22:26]

这个练习揭示了一个一致的规律：数学上的反向传播规则在教育型引擎和工业级引擎之间并不会改变。改变的是周围的机制——设备分发、向量化、类型分支和内存布局——但核心的链式法则公式保持不变。

PyTorch 还通过其 torch.autograd.Function API 展示了这种可扩展性。要注册一个新的原语操作，用户需要继承 Function 并实现两个静态方法：

class LegendrePolynomial3(torch.autograd.Function):
    @staticmethod
    def forward(ctx, input):
        ...

    @staticmethod
    def backward(ctx, grad_output):
        ...

只要提供了前向传播和局部反向传播的定义，PyTorch 就会自动将该操作集成到自动求导图中。它成为更大模型中的另一个乐高积木块，无需进一步手动干预即可通过它进行反向传播。[02:23:59]
